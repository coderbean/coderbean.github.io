幂等设计
2018-12-18 /
  • Java

我们实际系统中有很多操作，是不管做多少次，都应该产生一样的效果或返回一样的结果。 例如

  1. 前端重复提交选中的数据，应该后台只产生对应这个数据的一个反应结果；
  2. 我们发起一笔付款请求，应该只扣用户账户一次钱，当遇到网络重发或系统bug重发，也应该只扣一次钱；
  3. 发送消息，也应该只发一次
  4. 创建业务订单，一次业务请求只能创建一个，创建多个就会出大问题等等很多重要的情况都需要幂等的特性来支持。
